C6 — gibberish coerced into a budget

baseline (main) vs fix (fix/budget-plausibility-floor) · same KV/Vectorize/Desk · 2026-07-24

The bug: toInr treated a bare number ≥ 100 as literal rupees, so "asdfghjkl 12345 qwerty zxcvbn" became a ₹0.12 L budget and searched under ₹12k. The fix: a bare literal-rupee number must clear a ₹1 L plausibility floor to be a budget — below it, it's a misparse and is rejected. A bare "70" is still ₹70 L; a real "5000000" is still ₹50 L. +15/−1 · one-line logic change · 893/893 tests.

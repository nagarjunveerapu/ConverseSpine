Honest read: this A/B cannot isolate Stage 2's ~1s tail saving and it is wrong to read the means as a regression. The arms took different goal-paths per turn (LLM non-determinism) — baseline often landed on cheap paths (greet/commit, no search) while Stage 2 landed on heavy ones (recommend/answer with search) — so end-to-end time is dominated by path + DeepSeek variance (±sec), which swamps a 1s change. The tail saving is real but only visible in the per-stage measurement (tail = 1.0–1.6s, now off-path) and structurally (tail runs after reply+state are frozen, mutates no state). Lesson: never A/B a ~1s post-reply change end-to-end against a non-deterministic LLM pipeline.

  6. Budget-stretch honesty (C7) — not applied on a named-project pivot over budget.
